# Wiki Skill — 知识沉淀与维护

用于维护基于 Markdown 的长期知识库。目标是把可复用结论沉淀为可检索、可迭代、可审计的页面资产。

## 目标

- 不让高价值结论只停留在聊天记录里
- 让知识持续累积，而不是每次从零检索
- 让每次写入都有索引与日志可追踪

## 数据结构

```
$WORKSPACE_DIR/memory/
├── index.md               # 内容索引（按主题/页面组织）
├── log.md                 # 变更日志（按时间追加）
└── wiki/
    └── YYYY-MM-DD-*.md    # 单篇沉淀页面
```

## 强制禁止项

- 不要在未读取 `memory/index.md` 前全库扫读。
- 不要在证据不足时凭记忆补结论。
- 不要只改 `memory/wiki/*.md` 而漏改 `memory/index.md` 或 `memory/log.md`。
- 不要把同一主题反复新建页面，优先增量更新已有页面。
- 不要写入 workspace 之外路径。

## 失败信号

出现以下任一情况，说明流程偏离，应立即纠正：

- 回答没有引用具体页面路径。
- 读取范围过大，且没有先走 `index.md` 筛选。
- 页面已更新，但 `index.md` 或 `log.md` 未同步。
- 结论语气过强，但缺少依据或不确定性标注。
- 同一主题出现多个高度重叠页面，却没有合并判断。

## 固定检索顺序

1. 先读 `memory/index.md`，定位候选页面。
2. 仅读取高相关页面全文，不做无目的全库扫读。
3. 页面证据不足时，明确说明不确定，不强行下结论。
4. 当新增页面不再明显提升答案质量时停止扩读。

## 页面复用规则

- 同一主题已有页面时，优先增量更新，不重复新建。
- 仅当内容足以独立成主题时才创建新页。
- 避免以不同日期反复创建近似同题页面。
- 对稳定方法、标准决策、长期对比，优先收敛到单一规范页面。

## 何时触发沉淀

- 用户明确要求：`记一下`、`沉淀一下`、`整理成文档`、`写进知识库`
- 形成稳定方法、对比结论、标准清单
- 同类问题重复出现，值得形成标准页面

若同一回合同时命中 `todo/cases/insights` 与 wiki：

1. 有明确任务事项优先写 `todo`
2. 有具体事件复盘优先写 `cases`
3. 有个人体会优先写 `insights`
4. 可复用方法与结论同步写 `wiki`

## 工作流执行

### `ingest`

1. 从当前会话或输入材料提取可复用结论。
2. 先判断是更新现有页面还是新建页面。
3. 写入 `memory/wiki/YYYY-MM-DD-<slug>.md`。
4. 更新 `memory/index.md`（链接 + 一句话摘要；如主题摘要已变则同步刷新）。
5. 追加 `memory/log.md`（时间、动作、页面、原因）。

### `query`

1. 执行固定检索顺序。
2. 基于命中页面合成答案。
3. 在答案中标注引用页面路径。
4. 明确区分“已知事实”与“推断”。
5. 若证据不完整，明确标注不确定性。
6. 若用户明确要求沉淀，再将答案归档到 `memory/wiki/`。

### `lint`

定期体检并报告：

- 索引孤儿页（`memory/wiki/` 中存在但未被 `memory/index.md` 触达）
- 重复主题页（主题高度重叠且应优先合并）
- 页面已更新但索引或日志未同步
- 页面长期未更新且结论可能失效
- 页面结论强于依据，缺少足够支撑

### `update`

1. 更新目标页面内容。
2. 同步更新 `memory/index.md` 摘要（必要时）。
3. 追加 `memory/log.md` 记录变更原因与范围。

## 写入流程

1. 新建或更新 `memory/wiki/YYYY-MM-DD-<slug>.md`
2. 更新 `memory/index.md`
3. 追加 `memory/log.md`
4. 若是多文件写入，使用 `flock -x "$WORKSPACE_DIR/.memory.lock"` 加锁

## 页面模板

```markdown
# 标题

## 背景

## 结论

## 依据

## 后续动作
```

## 输出契约

- 先说明本次使用了哪些页面。
- 明确区分“已知事实”与“推断”。
- 证据不完整时明确标注不确定性。

## 约束

- 写入时尽量保留用户原意，不擅自改写结论方向
- 优先增量更新已有 wiki 页面，避免重复造页
- 仅依赖通用能力：读文件、检索、编辑、日志追加
- 所有路径必须在当前 workspace 内
