# Todo Skill — Task Management

This skill defines how to handle **reminders, additions, completion toggles, daily logs, and historical reviews**.

---

## Data File Structure

```
memory/
├── todos_today.md              # Today's tasks (reset daily)
├── todos_ongoing.md            # Ongoing tasks (long-term)
└── daily_logs/
    ├── YYYY-MM-DD.md           # Daily work log (generated on wrap-up)
    └── todos_YYYY-MM-DD.md     # Tasks scheduled for a future date
```

Paths are relative to the workspace root. Use `.memory.lock` when writing multiple files.

---

## Trigger Recognition

| User says | Action |
|-----------|--------|
| "what do I have today", "my todos", "remind me", "today's tasks" | → **Reminder** |
| "add", "write down", "new todo" | → **Add** |
| "done", "finished", "complete", "✓ XXX" | → **Toggle complete** |
| "wrap up", "done for the day", "generate log" | → **Daily log** |
| "last week", "yesterday", "review" | → **Review history** |

## Feature 1: Reminder

Read today's tasks and ongoing tasks, formatted for output. **Check for day rollover first.**

### Day Rollover Check

The first line of `todos_today.md` should be: `# Today's Todos — YYYY-MM-DD`

If the date doesn't match today:
1. Optionally ask whether to move unfinished items to ongoing
2. Write a new date header to `todos_today.md`
3. **Do not auto-clear** completed items; only clear when the user says "clear today's todos" or "new day"

### Output Format

```
📋 **Today's Todos** (YYYY-MM-DD)

**Today**
- [ ] Task A
- [x] Task B (completed)

**Ongoing**
- [ ] Long-term item C (in progress)

N items remaining.
```

---

## Feature 2: Add

Append a new item to today's todos, a future date, or ongoing tasks.

### Routing Rules

| User description | Target |
|------------------|--------|
| Contains "today" or no date | `memory/todos_today.md` |
| Specific date ("next Tuesday", "March 17", "tomorrow") | `memory/daily_logs/todos_YYYY-MM-DD.md` |
| Contains "ongoing", "long-term", "for a while" | `memory/todos_ongoing.md` |
| Unclear | Ask: "Is this for today, a specific day, or an ongoing item?" |

### Entry Format

```
- [ ] <description> (added YYYY-MM-DD)
```

### Example: Add to today's todos

```bash
flock -x .memory.lock -c "
  mkdir -p memory/daily_logs
  if [ ! -f memory/todos_today.md ] || ! head -1 memory/todos_today.md | grep -q '[0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\}'; then
    echo \"# Today's Todos — $(date +%Y-%m-%d)\" > memory/todos_today.md
  fi
  echo '- [ ] <description> (added $(date +%Y-%m-%d))' >> memory/todos_today.md
"
```

### Merge Future Tasks

If `memory/daily_logs/todos_YYYY-MM-DD.md` exists for today, merge and display it together with today's todos.

---

## Feature 3: Toggle Complete

Change `[ ]` to `[x]` or vice versa.

- "Finished XXX" / "XXX is done" → `[x]`
- "XXX is not done" / "undo complete" → `[ ]`
- If ambiguous, list current incomplete items for the user to choose

Search in both `memory/todos_today.md` and `memory/todos_ongoing.md`. Confirm after toggling.

---

## Feature 4: Daily Log

When the user says "wrap up" or asks for a log, summarize the day into a log file.

### Log File Path

`memory/daily_logs/YYYY-MM-DD.md`

### Log Template

```markdown
# Work Log — YYYY-MM-DD (Weekday)

## Today's Todo Completion

| Status | Item |
|--------|------|
| ✅ Done | Item A |
| ❌ Not done | Item B |

**Completion rate**: N/M (N%)

## Ongoing Status

- [ ] Long-term item C (in progress)

## Summary

> [3-5 sentences generated by AI based on completion]

---
_Logged at HH:MM_
```

### After Wrap-Up

1. Keep completed items in `todos_today.md` for next-day reference
2. **Do not auto-clear** `todos_today.md`

---

## Feature 5: Historical Review

Query and display past daily logs.

| User says | Resolution |
|-----------|------------|
| "What did I do yesterday" | Read `memory/daily_logs/YYYY-MM-DD.md` (yesterday) |
| "What did I do last week" | Read and summarize last week's logs |
| "What did I do on March 10" | Read `memory/daily_logs/2026-03-10.md` |
| "Review the last 7 days" | Read recent 7 logs and summarize |

---

## Notes

- Always use `flock -x .memory.lock` for file writes
- Create `memory/daily_logs/` on demand with `mkdir -p`
- Ongoing items are not affected by "clear today's todos"
