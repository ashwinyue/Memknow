文档首页

这里是 `Memknow` 的文档入口。最适合先读的是系统导览页，它会按主链路、workspace、session、长期记忆、heartbeat 和 schedule 把项目内部运作方式完整讲一遍。

进入系统导览页 查看项目 README

如果你是通过 GitHub Pages 打开这里，这个页面就是默认入口，不再是纯跳转页。

直接开始阅读

先看这条主链路

1. Feishu Receiver 接 WebSocket 事件，解析消息与附件。
2. Session Worker 按 channel_key 串行执行，组装上下文。
3. Claude Executor 在目标 workspace 中调用 Claude Code CLI。
4. Memory & Schedule 补齐跨 session 连续性与主动执行能力。

推荐阅读顺序

1. 系统导览页 2. README 3. design.md 4. quickstart.md

如果你是第一次接触项目，先看导览页建立整体心智模型；如果你准备部署，再看 quickstart 和 configuration。
